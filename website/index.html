Our primary objective in this project is be to develop a test suite with which to conveniently and accurately measure various versions of a measurement chip which has been in development at the ime for a few years.

While some measurements have already been performed, the amount of tests run has so far been limited due to time constraints by the respective teams. We aim to expand on those measurements with our test suite, as well as to use them as a reference point against which we can compare our findings.

The measuremet process has been largely automated. Both signal generators as well as the oscilloscope and the data acquisition process on the Raspberry Pi are controlled from a PC by means of Python and shell scripts. This allows convenient and error-resilient measuring of large data sets in comparatively little time (about five work-days for a full suite as run in our project).

Physical interfacing is achieved via various types of connections, as schematically indicated in the below graphic. The signal generators are connected to the PC via RS232 to USB adapters, while the oscilloscope's and the Raspberry Pi's connections to the PC are accomplished via Ethernet.

For interfacing with the sensor IC, Tobias Burgherr developed the below test board in his project. It provides all the necessary connections for the various supply voltages, output pins and enables convenient configuration of various settings of the chip (e.g. preamp gain).
